Paréntesis y momento de reflexión 🤔 (Patrocinado por Denis)¶

  • En la comununidad de Machine Learning existe la dicotoma de "Regresin vs. Clasificacin"
  • En realidad, es una simplificacin excesiva de todos los modelos y las herramientas que existen.
  • Regresin es un conjunto de herramientas, no un conjunto de problemas.
  • Por ejemplo: Regresin de Poisson no es continua (modela variable discreta), pero tampoco es clasificacin.
  • SIN EMBARGO esta dicotoma es tan bien conocida en la comunidad de ML que se la van a encontrar en entrevistas de trabajo, textos, conferencias, etc.

Modelos de clasificación¶

Se dividen en tres categoras:

  • Binarios
  • Multiclase (clasificar ms de dos clases)
  • Multietiqueta (un documento puede tener ms de una etiqueta)


Algunas medidas de rendimiento para modelos de clasificación¶

  • Exactitud $= \dfrac{\text{VP+VN}}{\text{VP}+\text{VN}+\text{FP}+\text{FN}} = \dfrac{\text{clasificado correcto}}{\text{nmero total de datos}}$
  • Sensibilidad $= \dfrac{\text{VP}}{\text{VP}+\text{FN}} = \dfrac{\text{VP}}{\text{lo que realmente es positivo}} $
  • Precisin $= \dfrac{\text{VP}}{\text{VP}+\text{FP}} = \dfrac{\text{VP}}{\text{lo que fue clasificado positivo}} $
  • F1 $= 2 \cdot \dfrac{\text{precisin}\cdot\text{sensibilidad}}{\text{precisin}+\text{sensibilidad}} $

Regresión Logística¶

Es el modelo ms famoso de Machine Learning despus de la regresin lineal

Mientras que la regresin lineal se usa para predecir/pronosticar valores continuos, la regresin logstica se usa habitualmente para clasificar valores discretos.

Las predicciones estn mapeados entre 0 y 1 a travs de la funcin logstica, lo que significa que las predicciones pueden ser interpretadas como probabilidades.

Regresión Logística¶

VENTAJAS:

  • Los resultados tienen una elegante representacin probabilistica
  • El modelo puede ser regularizado para evitar sobreajuste (overfitting)

DESVENTAJAS:

  • Tiende a tener un rendimiento inferior cuando hay multiples lineas de decisin o estas no son lineales
  • No es lo suficientemente flexible para capturar relaciones ms complejas

Máquinas de vectores de soporte (support vector machines - SVM)¶

Las MVS usan un mecanismo llamado kernel ( ncleo) que esencialmente calcula la distancia entre dos observaciones. Con esto, el modelo encuentra un lmite de decisin (hiperplano) ptimo para separar las clases.

Máquinas de vectores de soporte (support vector machines - SVM)¶

Una MVS con un ncleo lineal es muy similar a la regresin logstica, as que por lo general se usa con ncleos no-lineales con datos no-lineales


Máquinas de vectores de soporte (support vector machines - SVM)¶

VENTAJAS:

  • Puede modelar relaciones no lineales
  • Hay muchos ncleos de dnde escoger
  • Son buenos evitando sobreajuste

DESVENTAJAS:

  • Intensos con la memoria
  • Difcil de ajustar, difcil de eligir ncleo
  • No le va bien con grandes conjuntos de datos

Clasificador bayesiano ingenuo (Naive Bayes)¶


  • Modelo basado en el teorema de Bayes que explica la probabilidad condicional. El modelo se actualiza a travs de tus datos de entrenamiento
  • Se le llama "ingenuo" porque asume independencia condicional (todas los atrbutos son independientes)

Clasificador bayesiano ingenuo (Naive Bayes)¶

VENTAJAS:

  • Aunque la suposicin de independencia condicional rara vez es cierta, sorprendentemente tiene muy buenos resultados en la prctica
  • El modelo es fcil de implementar

DESVENTAJAS:

  • Debido a su simplicidad, aunque tiene buenos resultados, otros modelos por lo general tienen mejores resultados

Árboles de decisión¶


  • Es un algoritmo que puede ser usado tanto para clasificacin como para regresin.
  • La meta es crear un modelo que predice el valor de la variable objetivo aprendiendo simples reglas de decisin.

VENTAJAS:

  • Fcil de entender e interpretar.
  • No requiere mucha preparacin de los datos.

DESVENTAJAS:

  • Propensos al sobreajuste.
  • Son inestables, pero esto se puede solucionar con un ensamble.

import re
from nltk.corpus import stopwords
from sklearn.feature_extraction.text import TfidfVectorizer
stopwords_sp = stopwords.words('spanish')

def pre_procesado(texto):
    texto = texto.lower()
    texto = re.sub(r"[\W\d_]+", " ", texto)
    texto = texto.split() # Tokenizar
    texto = [palabra for palabra in texto if palabra not in stopwords_sp]
    texto = " ".join(texto)
    return texto

tfidf_vect = TfidfVectorizer(preprocessor=pre_procesado)
tfidf_vect.fit(train.mensaje.values);
